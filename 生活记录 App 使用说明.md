# 生活记录 App 使用说明

## 一、App 简介

生活记录是一个纯本地的个人日记/记录 App，帮助你按时间线记录日常生活中的点滴。支持自定义树状标签体系、属性标签、项目归类、文本格式、AI 助写、批量管理、主题切换、Excel 导出和备份恢复。

> **数据安全**：所有数据存储在手机本地（AI 助写除外）。可通过 JSON 导出+坚果云 WebDAV 实现多设备间手动同步。

---

## 二、首页（时间线）

打开 App 后看到的是**首页时间线**：

- 记录按**日期分组**显示，每天为一个卡片
- 点击日期标题行可展开/收起当天的记录
- 每条记录显示：**时间 + 标题/内容（最多 2 行）+ 项目名 + 树状标签 + 属性标签**
- 点击记录 → 进入**详情页**查看全文

### 顶部工具栏

**第一行（AppBar）**：

| 图标 | 功能 |
|------|------|
| 📋 | 进入**列表视图** |
| 🏷 | 进入**标签管理** |
| ✨ | 进入**属性标签管理** |
| 📁 | 进入**项目管理** |
| ⋮ | 更多菜单：导出 Excel / 数据备份 / 坚果云同步 / 切换入口 / AI 助写设置 / 主题 / 隐私锁 |

**第二行（展开/折叠栏）**：仅在有时记录显示。左侧「展开」「折叠」按钮控制所有日期分组的展开/折叠状态，右侧有 **↪ 撤销按钮**（灰色禁用，有操作时高亮可点）和 **↕ 排序按钮**（切换正序/倒序），以及记录总数。

### 底部 ⊕ 按钮

点击新建一条记录。

## 三、添加 / 编辑记录

### 新建记录

点击首页右下角的 **⊕** 按钮，弹出编辑器：

1. **点击「事项简介」** — 弹出输入框，支持多行（标题会加粗显示）
2. **点击「详细情况」** — 弹出大输入框，顶部有**格式化工具栏**
3. **选择树状标签**（可选）— 在标签树中选一条路径（单分支）
4. **选择属性标签**（可选）— 多选扁平标签
5. **选择项目**（可选）— 单选
6. 点击右上角 **「保存」**

### 格式化工具栏

内容输入框顶部的工具栏按钮：

| 按钮 | 插入语法 | 效果 |
|------|---------|------|
| **B** | `**文字**` | 粗体文字 |
| *I* | `*文字*` | 斜体文字 |
| • | `- 项目` | 无序列表 |
| 1. | `1. 项目` | 有序列表 |
| T | `# 标题` | 标题 |
| ✨ | （弹出风格菜单） | AI 润色 |

### AI 助写

点击 ✨ 按钮 → 选择写作风格 → AI 自动润色当前文本。支持以下风格：

| 风格 | 说明 |
|------|------|
| 用户原文风格 | 只修正语病，保留原味 |
| 正式书面风格 | 规范化书面语 |
| 跟自己聊天风格 | 口语化表达 |
| 鲁迅风格 | 冷峻凝练 |
| 徐志摩风格 | 优美诗意 |
| 文言文风格 | 翻译为文言文 |
| 自定义 | 用户自定提示词 |

AI 供应商和 API Key 在 `⋯` → **AI 助写设置** 中配置。输出长度不超过原文的 120%。

### 编辑已有记录

在首页或列表视图中点击记录 → 进入**详情页** → 右上角 ✎ 按钮 → 修改后保存。

## 四、详情页（卡片式展示）

页面分为 3 张卡片：

1. **元信息卡** — 时间、项目（橙色标签）、树状标签（紫色标签）、属性标签（灰色边框标签）
2. **标题卡** — 事项简介（如果有）
3. **内容卡** — 完整内容，**粗体**、*斜体*、列表、标题等格式已渲染

右上角 ✎ 按钮 → 编辑模式。编辑返回后列表自动刷新。

---

## 五、标签管理

### 树状标签

标签是**树状结构**，支持无限层级。例如：

```
生活
├── 饮食
│   ├── 早餐
│   ├── 午餐
│   └── 晚餐
├── 运动
│   ├── 跑步
│   └── 健身
└── 阅读
```

**操作**：
- **展开/收起**：点击标签左侧的 ▶/▼ 图标
- **全部展开/收起**：AppBar 中的按钮
- **添加子标签**：点击标签右侧的 ⊕
- **重命名/删除**：点击标签右侧的 ⋮
- **拖拽排序**：长按拖拽手柄调整同级标签顺序
- **移动层级**：⋮ → 移动，选择新的父节点

### 属性标签

扁平的属性标签体系，可按分组归类。用于标记心情、地点、优先级等维度。

## 六、项目管理

项目用于给记录归类。项目可以分组，一条记录通常对应一个项目。

- **添加项目/分组**：点击右下角 ⊕
- **重命名/删除**：点击项目右侧的 ⋮

---

## 七、列表视图

列表视图提供**筛选**、**分组**和**批量操作**功能。

### 筛选（AND 组合多选）

| 筛选维度 | 说明 |
|---------|------|
| 📅 日期 | 选择单日或时段 |
| 🏷 树状标签 | 多选，OR 逻辑 |
| ✨ 属性标签 | 多选，OR 逻辑 |
| 📁 项目 | 多选，OR 逻辑 |
| 🔍 关键词 | 搜索标题、内容、标签名 |

所有筛选条件用 **AND** 组合，同维度内用 **OR**。

### 分组

筛选栏右侧选择分组方式：按日期 / 按项目 / 按树状标签 / 按属性标签 / 不分组。
每组可单独展开/收起，分组栏也有「全部展开/折叠」按钮。

### 单条删除

非选择模式下每条记录右侧有 🗑 图标，点击确认删除。删除后可撤销。

### 批量操作

1. 点击右上角 ✓ 进入选择模式
2. 勾选记录
3. 工具栏操作：替换树状标签 / 替换属性标签 / 设置项目 / 批量删除

## 八、文本格式

支持以下格式（在内容中直接输入）：

| 语法 | 效果 |
|------|------|
| `**文字**` | **粗体文字** |
| `*文字*` | *斜体文字* |
| `- 项目` | • 无序列表项 |
| `1. 项目` | 1. 有序列表项 |
| `# 标题` | 标题文字 |

编辑器有格式化工具栏辅助插入，详情页自动渲染。

## 九、隐私锁

首页 `⋯` → **隐私锁** → 开启后可设置：

| 设置项 | 说明 |
|--------|------|
| 启用隐私锁 | 开/关 |
| 面容/指纹解锁 | 支持 Face ID 的设备可优先使用生物识别 |
| 解锁密码 | 4-6 位密码 |

开启后，启动 App 或从后台切换回来时自动锁定，需要验证身份才能使用。

> **设置持久化**：所有设置（AI Key、隐私锁、主题等）均保存在本地数据库中，重启 App 不会丢失。

## 十一、坚果云同步

首页 `⋯` → **坚果云同步**，可配置 WebDAV 在多设备间手动同步数据。

### 第一次配置

1. 在坚果云官网 → 安全选项 → **添加第三方应用密码**
2. App 内填写：**WebDAV 地址**（`https://dav.jianguoyun.com/dav/`）、**邮箱**、**应用密码**
3. 点击 **「测试连接并保存」**

### 上传到云端

- 点击 **「上传到坚果云」**，将当前所有数据导出为 JSON 并上传
- 每次上传生成新文件（带时间戳），不会覆盖旧备份

### 从云端恢复

1. 点击 **「刷新云端备份列表」** 查看云端文件
2. 选择需要恢复的备份 → **「恢复」**
3. ⚠️ 恢复会替换本地所有数据，建议先上传一份当前备份

### 跨设备同步流程

```
电脑端记了内容 → 点「上传到坚果云」
  → 手机端点「刷新」→ 看到备份文件 → 点「恢复」
  → 手机本地数据 = 电脑端上传时的内容
```

## 十二、数据迁移（备份与恢复）

### 导出 JSON 备份

1. 首页 `⋯` → **数据备份**
2. 点击 **「导出备份」**
3. 文件保存到 App 文档目录
4. 可点击 **「分享」** 发到微信 / AirDrop

### 导入 JSON 恢复

**方式一：从 iPhone「文件」App 选择**
- 点击 **「从 iPhone 文件 App 选择」**
- 在文件浏览器中选中 `.json` 备份文件

**方式二：扫文档目录**
- 通过爱思助手 → 文件共享 → 将 `.json` 放入 App 的 Documents 目录
- 点击 **「扫描此路径」** → 选择备份文件 → **「恢复」**

> ⚠️ **恢复会替换全部现有数据，不可撤销！建议先导出备份。**

1. GitHub Actions 下载 `.ipa`（项目主页 → Actions → 最新成功构建 → Artifacts）

## 十五、常见问题

**Q: 数据存在哪里？**
本地 SQLite 数据库。AI 助写会发送文本到第三方 API。

**Q: 导入 JSON 找不到文件？**
可用「从 iPhone 文件 App 选择」直接选取，或用爱思助手放入 Documents 目录。

**Q: 怎么把 Excel 弄到电脑上？**
导出后点击「打开」→ 系统分享菜单 → 微信 / AirDrop / 邮件。

**Q: AI 需要什么 Key？**
推荐 DeepSeek（platform.deepseek.com），也支持通义千问、Claude、OpenAI 等。
